# Components

Reusable standalone slide building blocks.

Current items:

- `badge/` — compact labels.
- `callout/` — highlighted note, warning, insight, or takeaway.
- `card/` — flat feature/explanation cards.
- `code-block/` — presentation-ready code snippets without a syntax highlighter dependency.
- `code-diff/` — compact static diffs for implementation change explanations.
- `divider/` — internal separator for dense slides.
- `file-tree/` — static file/folder trees for project structure slides.
- `http-exchange/` — static request/response blocks for API walkthroughs.
- `image-card/` — image or case-study cards with captions.
- `metric/` — single KPI/stat block.
- `panel/` — flexible visual container for grouped content.
- `progress/` — accessible progress bars/meters.
- `quote/` — testimonial, pull quote, or cited statement.
- `table/` — semantic presentation table styling.
- `terminal/` — static terminal-style command and output blocks.
- `timeline/` — reusable sequence component for milestones and processes.

Components depend on `core/base`, should work anywhere, and should not require a specific slide layout parent.
